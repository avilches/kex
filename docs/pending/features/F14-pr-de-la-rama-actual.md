## Contexto

Dada la rama git del panel activo, resolver el Pull Request de GitHub asociado y
devolver lo minimo para pintar un badge: numero, estado (open / merged / closed)
y URL. Es la pieza atomica que pide el punto 3 de la seccion "Workspace: label de
texto + barra superior con contexto" en `docs/TODO.md` ("PR de la rama actual"),
y es la base tecnica reutilizable de [F13](F13-workspace-mostrar-prs-repo.md)
(lista de PRs del repo): F13 lista; F14 resuelve el de la rama activa.

El diseno replica como lo hace cmux (terminal-app de referencia), que combina dos
vias: el `gh` CLI cuando esta disponible y la REST API de GitHub como respaldo.
Aqui se acota al alcance minimo (number, state, url) y al backend Rust, dejando la
presentacion (badge en barra superior) para una iteracion de UI posterior.

### Como lo hace cmux (referencia)

- **Via shell (gh):** integracion de shell que, al cambiar de rama, ejecuta
  `gh pr view <branch> --json number,state,url` y reporta el resultado por su
  socket. Cachea en disco con TTL ~45 s y salta `main`/`master`.
- **Via app (REST):** servicio de polling que llama a
  `GET /repos/{owner}/{repo}/pulls?state=all&sort=updated&per_page=100` (hasta 2
  paginas) y, si no encuentra la rama, a
  `GET /repos/{owner}/{repo}/pulls?head={owner}:{branch}&state=all`. Autentica con
  `Authorization: Bearer <token>` tomado de `GH_TOKEN`/`GITHUB_TOKEN` o, en su
  defecto, de `gh auth token`. Cachea por repo ~15 s y prioriza el PR OPEN mas
  reciente.
- **Mapeo rama -> repo:** parsea `git remote -v` para sacar `owner/repo`, con
  prioridad `upstream` > `origin`.

## Decisiones tomadas

- **Ambas vias con fallback.** Primero `gh pr view` (autoritativo si `gh` esta
  instalado y autenticado); si `gh` no esta disponible o falla por entorno, caer a
  la REST API directa.
- **Capa: backend Rust** como una IPC nueva `git_current_pr`, dentro del modulo
  `src-tauri/src/modules/git/`, reutilizando su infraestructura (runner con
  soporte WSL, resolucion de repo/remote, tipos serde, errores).
- **Cliente HTTP: `ureq`** (sincrono, ligero, rustls) para la via REST. Encaja con
  el patron `spawn_blocking` del modulo git y no obliga a ampliar las features de
  `tokio` (hoy solo `rt`). Trade-off: `ureq` + rustls anade peso de binario; es la
  opcion TLS-capaz mas ligera. La alternativa sin dependencia nueva (`gh api`) se
  descarto porque deja de ser un fallback real cuando `gh` no esta.
- **Datos minimos:** `number`, `state` (open/merged/closed), `url`. Sin title,
  checks ni autor en esta feature (eso vive en F13).
- **On-demand, nunca en bucle.** El comando se invoca bajo demanda desde el
  frontend (al cambiar de panel/rama, con debounce). Cachear y refrescar bajo
  demanda en el frontend; respetar la filosofia de perf del fork.

### Flujo de la orquestacion

1. `authorized_repo_root(registry, repo_root, workspace)` + `ensure_git_available`.
2. Resolver `branch`: usar el parametro si viene no vacio; si no, derivarlo de
   `operations::resolve_repo` (campo `branch`).
3. `should_skip_branch(&branch)` -> `Ok(None)` para `main`/`master`.
4. Resolver slugs `owner/repo`: leer `remote.upstream.url` y `remote.origin.url`
   (via `operations::remote_url`), parsear con `github_slug_from_remote_url`,
   dedupe preservando prioridad `upstream` > `origin`.
5. **Via A (gh):** por cada slug, `try_gh`. `Found(pr)` -> return. `NoPr` -> seguir.
   `Unavailable` -> marcar fallback.
6. Si `gh` resolvio limpiamente (nunca `Unavailable`) -> `Ok(None)` (gh es
   autoritativo, no se consulta REST).
7. **Via B (REST):** solo si hubo `Unavailable`. Obtener token
   (`GH_TOKEN`/`GITHUB_TOKEN` o `gh auth token`). Por cada slug, `try_rest`.
8. `Ok(None)` si nada resuelve.

### Detalle REST (via B)

- Endpoint: `https://api.github.com/repos/{slug}/pulls?head={owner}:{branch}&state=all&per_page=100`.
- Headers: `Authorization: Bearer <token>`, `Accept: application/vnd.github+json`,
  `User-Agent: kex-pr-probe`, `X-GitHub-Api-Version: 2022-11-28`.
- `ureq` con timeout 8 s; leer cuerpo con `.into_string()` y parsear con
  `serde_json` (ya presente) a `Vec<RestPr>`.
- `RestPr` (interno, `Deserialize`): `number: u64`, `state: String`,
  `html_url: String`, `merged_at: Option<String>`, `updated_at: Option<String>`,
  `head: { r#ref: String }` (con `#[serde(rename = "ref")]`).
- Seleccion `select_preferred`: preferir `OPEN`; entre varios, el `updated_at` mas
  reciente; si ninguno abierto, el mas reciente de los cerrados/merged.
- Mapear estado: `merged_at` presente -> `Merged`; si no, `open`/`closed`.

## Criterios de aceptacion

- Para una rama con PR abierto en un repo de GitHub, `git_current_pr` devuelve
  `{ number, state: "open", url }`.
- Con `gh` instalado y autenticado, la via A resuelve sin tocar la REST API.
- Sin `gh` (o `gh` no autenticado) pero con `GH_TOKEN`/`GITHUB_TOKEN`, la via B
  resuelve por REST.
- `main`/`master` no consultan nada y devuelven `None`.
- Sin remote de GitHub, sin red o sin credenciales, degrada a `None` o `Err`
  tipado, sin panico y sin colgar (timeouts respetados).

```rust
pub(crate) fn github_slug_from_remote_url(url: &str) -> Option<String> {
    const PREFIXES: [&str; 5] = [
        "git@github.com:",
        "ssh://git@github.com/",
        "https://github.com/",
        "http://github.com/",
        "git://github.com/",
    ];
    let trimmed = url.trim();
    let path = PREFIXES
        .iter()
        .find_map(|p| trimmed.strip_prefix(p))?;
    let mut parts = path.trim_start_matches('/').split('/');
    let owner = parts.next().filter(|s| !s.is_empty())?;
    let repo_raw = parts.next().filter(|s| !s.is_empty())?;
    let repo = repo_raw.strip_suffix(".git").unwrap_or(repo_raw);
    if repo.is_empty() {
        return None;
    }
    Some(format!("{owner}/{repo}"))
}
```

```rust
fn parse_state(raw: &str, merged_at: Option<&str>) -> PullRequestState {
    if merged_at.is_some() {
        return PullRequestState::Merged;
```

```rust
#[derive(Deserialize)]
struct RestRef {
    #[serde(rename = "ref")]
    r#ref: String,
}

#[derive(Deserialize)]
struct RestPr {
    number: u64,
    state: String,
    html_url: String,
    merged_at: Option<String>,
    updated_at: Option<String>,
    head: RestRef,
}

fn select_preferred(items: Vec<RestPr>) -> Option<RestPr> {
    let valid: Vec<RestPr> = items
        .into_iter()
        .filter(|p| p.number > 0 && !p.html_url.is_empty())
        .collect();
    let open_latest = valid
        .iter()
        .enumerate()
        .filter(|(_, p)| p.state.eq_ignore_ascii_case("open"))
        .max_by(|a, b| a.1.updated_at.cmp(&b.1.updated_at))
        .map(|(i, _)| i);
    let idx = open_latest.or_else(|| {
        valid
            .iter()
            .enumerate()
            .max_by(|a, b| a.1.updated_at.cmp(&b.1.updated_at))
            .map(|(i, _)| i)
    })?;
    valid.into_iter().nth(idx)
}

fn rest_pulls_endpoint(slug: &str, branch: &str) -> Option<String> {
    let owner = slug.split('/').next().filter(|s| !s.is_empty())?;
    Some(format!(
        "https://api.github.com/repos/{slug}/pulls?head={owner}:{branch}&state=all&per_page=100"
    ))
}
```

Nota: `head.r#ref` queda disponible para futuras validaciones de rama exacta; en
esta version el filtro `head=` ya acota por rama en el servidor.

```rust
    Found(PullRequestInfo),
    NoPr,
    Unavailable,

/// Runs `gh` capturing stdout/stderr with a timeout. Mirrors the git runner:
/// no stdin, hidden console on Windows, WSL passthrough when the workspace is WSL.
fn run_gh(
    workspace: &WorkspaceEnv,
    cwd: &str,
    args: &[&str],
    timeout_secs: u64,
) -> std::io::Result<(Option<i32>, String, String, bool)> {
    let mut cmd = build_gh_command(workspace, cwd, args);
    cmd.env("GH_PROMPT_DISABLED", "1")
        .env("GH_NO_UPDATE_NOTIFIER", "1")
        .env("CLICOLOR", "0")
        .stdin(Stdio::null())
        .stdout(Stdio::piped())
        .stderr(Stdio::piped());
    crate::modules::proc::hide_console(&mut cmd);

    let child = Arc::new(SharedChild::spawn(&mut cmd)?);
    let mut out_pipe = child.take_stdout().expect("stdout pipe");
    let mut err_pipe = child.take_stderr().expect("stderr pipe");
    let out_handle = thread::spawn(move || {
        let mut s = String::new();
        let _ = out_pipe.read_to_string(&mut s);
        s
    });
    let err_handle = thread::spawn(move || {
        let mut s = String::new();
        let _ = err_pipe.read_to_string(&mut s);
        s
    });

    let (tx, rx) = mpsc::channel();
    let waiter = Arc::clone(&child);
    thread::spawn(move || {
        let _ = tx.send(waiter.wait());
    });

    let (exit_code, timed_out) =
        match rx.recv_timeout(Duration::from_secs(timeout_secs.clamp(1, 30))) {
            Ok(Ok(status)) => (status.code(), false),
            Ok(Err(e)) => return Err(e),
            Err(mpsc::RecvTimeoutError::Timeout) => {
                let _ = child.kill();
                let _ = child.wait();
                (None, true)
            }
            Err(mpsc::RecvTimeoutError::Disconnected) => (None, true),
        };
    let stdout = out_handle.join().unwrap_or_default();
    let stderr = err_handle.join().unwrap_or_default();
    Ok((exit_code, stdout, stderr, timed_out))
}

#[cfg(windows)]
fn build_gh_command(workspace: &WorkspaceEnv, cwd: &str, args: &[&str]) -> Command {
    if let WorkspaceEnv::Wsl { distro } = workspace {
        let mut cmd = Command::new("wsl.exe");
        cmd.arg("-d").arg(distro);
        if !cwd.is_empty() {
            cmd.arg("--cd").arg(cwd);
        }
        cmd.arg("--exec").arg("gh").args(args);
        return cmd;
    }
    let mut cmd = Command::new("gh");
    cmd.args(args);
    if !cwd.is_empty() {
        cmd.current_dir(cwd);
    }
    cmd
}

fn try_gh(resolved: &ResolvedGitDirectory, slug: &str, branch: &str) -> GhOutcome {
    let args = [
        "pr", "view", branch, "--repo", slug, "--json", "number,state,url",
    ];
    match run_gh(&resolved.workspace, &resolved.git_path, &args, 10) {
        Ok((Some(0), stdout, _, false)) => {
            parse_gh_json(&stdout).map(GhOutcome::Found).unwrap_or(GhOutcome::Unavailable)
        }
        Ok((_, _, stderr, false)) if gh_output_indicates_no_pr(&stderr) => GhOutcome::NoPr,
        // gh present but errored (auth, rate limit, etc.) or timed out -> let REST try.
        Ok(_) => GhOutcome::Unavailable,
        // gh not installed (spawn error).
        Err(_) => GhOutcome::Unavailable,
    }
}
```

```rust
fn github_token(workspace: &WorkspaceEnv, cwd: &str) -> Option<String> {
    if let Ok(t) = std::env::var("GH_TOKEN") {
        if !t.trim().is_empty() {
            return Some(t.trim().to_string());
        }
    }
    if let Ok(t) = std::env::var("GITHUB_TOKEN") {
        if !t.trim().is_empty() {
            return Some(t.trim().to_string());
        }
    }
    // Fallback: ask gh for a token (works even when env is empty).
    match run_gh(workspace, cwd, &["auth", "token"], 5) {
        Ok((Some(0), stdout, _, false)) => {
            let t = stdout.trim().to_string();
            if t.is_empty() { None } else { Some(t) }
        }
        _ => None,
    }
}

fn try_rest(slug: &str, branch: &str, token: &str) -> Result<Option<PullRequestInfo>> {
    let Some(endpoint) = rest_pulls_endpoint(slug, branch) else {
        return Ok(None);
    };
    let resp = ureq::get(&endpoint)
        .timeout(Duration::from_secs(8))
        .set("Authorization", &format!("Bearer {token}"))
        .set("Accept", "application/vnd.github+json")
        .set("User-Agent", "kex-pr-probe")
        .set("X-GitHub-Api-Version", "2022-11-28")
        .call();
    let body = match resp {
        Ok(r) => r.into_string().unwrap_or_default(),
        // 4xx/5xx and transport errors degrade to "no PR".
        Err(_) => return Ok(None),
    };
    let items: Vec<RestPr> = serde_json::from_str(&body).unwrap_or_default();
    let Some(best) = select_preferred(items) else {
        return Ok(None);
    };
    Ok(Some(PullRequestInfo {
        number: best.number,
        state: parse_state(&best.state, best.merged_at.as_deref()),
        url: best.html_url,
    }))
}
```

```rust
fn repo_slugs(
    registry: &WorkspaceRegistry,
    repo_root: &str,
    workspace: &WorkspaceEnv,
) -> Vec<String> {
    let mut slugs: Vec<String> = Vec::new();
    for remote in ["upstream", "origin"] {
        if let Ok(Some(url)) = operations::remote_url(registry, repo_root, remote, workspace) {
            if let Some(slug) = github_slug_from_remote_url(&url) {
                if !slugs.contains(&slug) {
                    slugs.push(slug);
                }
            }
        }
    }
    slugs
}

pub fn current_pull_request(
    registry: &WorkspaceRegistry,
    repo_root: &str,
    branch: Option<String>,
    workspace: &WorkspaceEnv,
) -> Result<Option<PullRequestInfo>> {
    let resolved = authorized_repo_root(registry, repo_root, workspace)?;
    ensure_git_available(&resolved.workspace)?;

    let branch = match branch {
        Some(b) if !b.trim().is_empty() => b.trim().to_string(),
        _ => match operations::resolve_repo(registry, repo_root, workspace)? {
            Some(info) if !info.is_detached => info.branch,
            _ => return Ok(None),
        },
    };
    if should_skip_branch(&branch) {
        return Ok(None);
    }

    let slugs = repo_slugs(registry, repo_root, workspace);
    if slugs.is_empty() {
        return Ok(None);
    }

    // Via A: gh is authoritative when available.
    let mut gh_unavailable = false;
    for slug in &slugs {
        match try_gh(&resolved, slug, &branch) {
            GhOutcome::Found(pr) => return Ok(Some(pr)),
            GhOutcome::NoPr => {}
            GhOutcome::Unavailable => gh_unavailable = true,
        }
    }
    if !gh_unavailable {
        return Ok(None);
    }

    // Via B: REST fallback.
    let Some(token) = github_token(workspace, &resolved.git_path) else {
        return Ok(None);
    };
    for slug in &slugs {
        if let Some(pr) = try_rest(slug, &branch, &token)? {
            return Ok(Some(pr));
        }
    }
    Ok(None)
}
```
