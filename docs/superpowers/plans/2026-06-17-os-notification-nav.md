# OS Notification Navigation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** When a coding-agent OS notification is clicked, the correct Kex window gains focus and navigates to the workspace and panel where the agent is running.

**Architecture:** The frontend stores a pending navigation target in Rust before sending the OS notification. When any main window gains focus (which happens on notification click), Rust consumes the pending target, focuses the correct window, and emits `kex:activate-panel` to it. Each main window listens for that event and navigates internally. Two existing bugs are fixed in the same pass: the panel-visibility check that suppresses notifications incorrectly, and the missing terminal focus after navigation.

**Tech Stack:** Rust (Tauri 2), TypeScript (React 19, Zustand), `tauri-plugin-notification` (already a dep, frontend + Rust sides), vitest

```rust
use std::sync::Mutex;
use std::time::{Duration, Instant};

const NAV_TTL: Duration = Duration::from_secs(5);

pub struct PendingNav {
    pub window_label: String,
    pub workspace_id: String,
    pub panel_id: String,
    queued_at: Instant,
}

#[derive(Default)]
pub struct PendingNavState(Mutex<Option<PendingNav>>);

impl PendingNavState {
    pub fn store(&self, window_label: String, workspace_id: String, panel_id: String) {
        *self.0.lock().unwrap() = Some(PendingNav {
            window_label,
            workspace_id,
            panel_id,
            queued_at: Instant::now(),
        });
    }

    /// Takes the pending nav only if it was stored within the last 5 seconds.
    /// Stale or absent entries are discarded and None is returned.
    pub fn take_if_fresh(&self) -> Option<PendingNav> {
        let mut guard = self.0.lock().unwrap();
        match &*guard {
            Some(n) if n.queued_at.elapsed() < NAV_TTL => guard.take(),
            _ => {
                *guard = None;
                None
            }
        }
    }
}

#[tauri::command]
pub fn agent_queue_nav(
    state: tauri::State<'_, PendingNavState>,
    window_label: String,
    workspace_id: String,
    panel_id: String,
) {
    state.store(window_label, workspace_id, panel_id);
}

WindowEvent::Focused(true) | WindowEvent::Resized(_) => {
    let mgr = app_handle.state::<window_state::WindowStateManager>();
    if matches!(event, WindowEvent::Focused(true)) {
        mgr.set_focused_window(&win_label);
        // Consume any pending OS-notification navigation and route to the target window.
        // Only main windows (label starts with "w-") trigger this; settings window is ignored.
        if win_label.starts_with("w-") {
            let nav_state = app_handle.state::<agent::PendingNavState>();
            if let Some(nav) = nav_state.take_if_fresh() {
                if let Some(target) = app_handle.get_webview_window(&nav.window_label) {
                    let _ = target.set_focus();
                    let _ = target.emit("kex:activate-panel", serde_json::json!({
                        "workspaceId": nav.workspace_id,
                        "panelId": nav.panel_id,
                    }));
                }
            }
        }

import { invoke } from "@tauri-apps/api/core";
import { getCurrentWindow } from "@tauri-apps/api/window";
import {
  isPermissionGranted,
  requestPermission,
  sendNotification,
} from "@tauri-apps/plugin-notification";

let granted = false;

async function ensurePermission(): Promise<boolean> {
  if (granted) return true;
  let ok = await isPermissionGranted();
  if (!ok) ok = (await requestPermission()) === "granted";
  granted = ok;
  return ok;
}

export async function queueNavAndNotify({
  workspaceId,
  panelId,
  title,
  body,
}: {
  workspaceId: string;
  panelId: string;
  title: string;
  body: string;
}): Promise<void> {
  const windowLabel = getCurrentWindow().label;
  try {
    await invoke("agent_queue_nav", { windowLabel, workspaceId, panelId });
  } catch (e) {
    console.warn("[kex] agent_queue_nav failed:", e);
  }
  try {
    if (await ensurePermission()) sendNotification({ title, body });
  } catch (e) {
    console.warn("[kex] os notification failed:", e);
  }
}
```

import { usePreferencesStore } from "@/modules/settings/preferences";
import { showAgentToast } from "../components/AgentToast";
import { useAgentStore } from "../store/agentStore";
import { queueNavAndNotify } from "./notify";
import type { AgentSource, NotificationKind } from "./types";

type RouteArgs = {
  source: AgentSource;
  agent: string;
  kind: NotificationKind;
  title: string;
  body?: string;
  focused: boolean;
  /** True when the user is currently looking at this agent's panel. */
  visible: boolean;
  /** Allow an in-app toast when focused but not looking at the agent. */
  allowToast: boolean;
  tabId?: string;
  panelId?: string;
  onActivate: () => void;
};

export function routeAgentNotification({
  source,
  agent,
  kind,
  title,
  body,
  focused,
  visible,
  allowToast,
  tabId = "",
  panelId = "",
  onActivate,
}: RouteArgs): void {
  if (!usePreferencesStore.getState().agentNotifications) return;
  if (focused && visible) return;

  useAgentStore.getState().pushNotification({ source, agent, kind, tabId, panelId });

  if (!focused) {
    void queueNavAndNotify({ workspaceId: tabId, panelId, title, body: body ?? agent });
    return;
  }
  if (allowToast) {
    showAgentToast({ agent, title, body, onActivate });
  }
}

## Known Limitations

- If two agents finish while the user is away, only the most recent navigation target is stored; clicking either OS notification navigates to the last one. This is acceptable for now.
- TTL is 5 seconds: if the user focuses any Kex window within 5 seconds of a notification (for any unrelated reason), the nav fires. Increasing the TTL would increase the accidental-navigation window.
- On Linux and Windows, OS notification click behavior depends on the desktop environment. The nav mechanism (store + focus event) works the same way but the notification click may not reliably bring the app to focus on all DEs.
