# Editor Settings Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add per-extension visual toggles (line numbers, whitespace, fold gutter, word wrap) surfaced in the editor overlay bar, plus global editor settings (indentation, scroll past end, active line, editing toggles) and cursor configuration (editor + terminal), all persisted in settings JSON.

**Architecture:** Per-extension view settings live in a new `editorViewByExt` map in the settings store, resolved against prose/code default profiles by a pure function. CodeMirror features are wired through `Compartment`s (the established pattern) so they reconfigure at runtime. The overlay bar exposes the per-ext toggles via a `[...]` dropdown of checkbox items. Global editor + cursor settings follow the existing store pref + Settings `SettingRow` pattern. Word wrap migrates from the per-panel override model to the per-extension model.

**Tech Stack:** React 19 + TypeScript, CodeMirror 6 (`@uiw/react-codemirror`, `@codemirror/view`, `@codemirror/state`, `@codemirror/language`), xterm.js, shadcn/ui (`DropdownMenu`, `Switch`, `Select`), Zustand (`usePreferencesStore`), `tauri-plugin-store`, Vitest.

```ts
export type EditorViewSettings = {
  wrap: boolean;
  lineNumbers: boolean;
  whitespace: boolean;
  foldGutter: boolean;
};

export type EditorViewMap = Record<string, Partial<EditorViewSettings>>;

const PROSE_EXTS = new Set(["md", "markdown", "mdx", "txt", "text"]);

const PROSE_DEFAULTS: EditorViewSettings = {
  wrap: true,
  lineNumbers: false,
  whitespace: false,
  foldGutter: false,
};

const CODE_DEFAULTS: EditorViewSettings = {
  wrap: false,
  lineNumbers: true,
  whitespace: false,
  foldGutter: true,
};

export function extOf(path: string): string {
  const base = path.split(/[\\/]/).pop() ?? "";
  const dot = base.lastIndexOf(".");
  if (dot <= 0) return "";
  return base.slice(dot + 1).toLowerCase();
}

export function defaultsForExt(ext: string): EditorViewSettings {
  return PROSE_EXTS.has(ext) ? { ...PROSE_DEFAULTS } : { ...CODE_DEFAULTS };
}

export function resolveEditorView(
  path: string,
  map: EditorViewMap,
): EditorViewSettings {
  const ext = extOf(path);
  return { ...defaultsForExt(ext), ...(map[ext] ?? {}) };
}
```

Note: `.gitignore` -> `extOf` gives `"gitignore"` because `lastIndexOf(".")` is index 0 only for a leading dot with nothing before it; the `dot <= 0` guard makes a leading-dot-only name return `""`, but `.gitignore` has the dot at index 0 and content after, so it returns `"gitignore"`. The test above asserts exactly this.

### Task 5: Editor cursor extension (blink + style)

- [ ] **Step 1: Implement**

```ts
import { type Extension } from "@codemirror/state";
import { EditorView } from "@codemirror/view";
import type { CursorStyle } from "@/modules/settings/store";

// CodeMirror's default caret blinks (~1.2s). Disable it by killing the
// animation on the cursor layer; enabling just restores the default.
export function cursorBlinkExt(blink: boolean): Extension {
  if (blink) return [];
  return EditorView.theme({
    ".cm-cursorLayer": { animation: "none !important" },
  });
}

// The editor is monospace, so a fixed 1ch width aligns the block/underline
// caret with the glyph cell. Translucent fill keeps the character readable,
// matching the vim fat-cursor look. Vim manages its own caret, so skip there.
export function cursorStyleExt(
  style: CursorStyle,
  vimActive: boolean,
): Extension {
  if (vimActive || style === "bar") return [];
  if (style === "block") {
    return EditorView.theme({
      ".cm-cursor.cm-cursor-primary": {
        width: "1ch",
        borderLeft: "none",
        backgroundColor:
          "color-mix(in srgb, var(--foreground) 35%, transparent)",
        outline:
          "1px solid color-mix(in srgb, var(--foreground) 55%, transparent)",
      },
    });
  }
  return EditorView.theme({
    ".cm-cursor.cm-cursor-primary": {
      width: "1ch",
      borderLeft: "none",
      borderBottom: "2px solid var(--foreground)",
    },
  });
}
```
