# Global Naming Refactor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give every core concept exactly one canonical name across variables, methods, types, JSON keys, IPC commands, and ID prefixes.

**Architecture:** This is a pure rename refactor with full existing test coverage. The safety net is the compiler and the existing suite, not new tests. Each task performs one logical rename, then must leave `pnpm check-types`, `pnpm exec biome lint ./src`, `pnpm test`, and (when Rust changes) `cargo clippy` + `cargo test --locked` green. Phases are strictly ordered: Phase 1 frees the word "tab" from its five non-entity meanings, Phase 2 renames the `Panel` entity to `Tab` (now that `tabId` no longer means a workspace id), Phase 3 cleans up secondary inconsistencies.

**Tech Stack:** Tauri 2 + Rust, React 19 + TypeScript, Zustand, tauri-plugin-store, Biome (lint), Vitest (`pnpm test`).

## Global Constraints

- No backward compatibility: rename JSON keys, IPC commands, and ID prefixes freely. Write no migration code. Remove pre-existing migration shims that this refactor makes pointless.
- Commits atomic (one logical rename per commit), messages in English, no em-dash, no emoji, no "Co-authored-by" / "Generated by Claude Code".
- Frontend imports always `@/...`, never relative across modules. pnpm only.
- Lint runs directly via Biome, never `pnpm lint` (RTK rewrites it to ESLint and fails): use `pnpm exec biome lint ./src`.
- All entity IDs come from `src/lib/ids.ts` (`nid()`); never `crypto.randomUUID()` / `Math.random()`.
- Living docs updated in the SAME commit as the code: `docs/ARCHITECTURE.md`, `docs/IPC.md`, `docs/WORKSPACES.md`, `docs/WORKSPACES_GOTCHAS.md`, `AGENTS.md`, `CLAUDE.md`.
- Spec: `docs/superpowers/specs/2026-06-30-naming-refactor-design.md`.
