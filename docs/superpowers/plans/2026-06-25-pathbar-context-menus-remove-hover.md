# Path bar context menus + remove tab HoverCard Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give the terminal the editor's badge breadcrumb, add per-segment and per-file context menus to both path bars, route file mutations to the explorer, and delete the floating tab HoverCard (moving its content into a terminal `[...]` menu).

**Architecture:** Extract a shared `PathBreadcrumb` from the editor's breadcrumb; both path bars consume it. Directory-segment and file-leaf context menus reuse the explorer's existing handlers via shared menu-item definitions and an extended `RevealRequest` channel (`pendingAction`) that makes the explorer perform rename/duplicate/delete after revealing. The terminal's right side gains inline indicators (run-on-start icon, agent chip) plus a `[...]` dropdown that absorbs the HoverCard's run-on-start / session / transcript / started content.

**Tech Stack:** React 19 + TypeScript, shadcn/ui (`ContextMenu`, `DropdownMenu`, `Breadcrumb`, `Badge`), hugeicons, Tailwind v4, Vitest.

```ts
export type RevealAction = "rename" | "duplicate" | "delete";

export function dispatchRevealAction(
  action: RevealAction,
  file: string,
  isDir: boolean,
  handlers: {
    beginRename: (path: string) => void;
    beginDuplicate: (path: string, kind: "file" | "dir") => void;
    requestDelete: (path: string, isDir: boolean) => void;
  },
): void {
  if (action === "rename") handlers.beginRename(file);
  else if (action === "duplicate") handlers.beginDuplicate(file, isDir ? "dir" : "file");
  else if (action === "delete") handlers.requestDelete(file, isDir);
}
```
