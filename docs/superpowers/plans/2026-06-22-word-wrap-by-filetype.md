# Word Wrap By File Type Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make editor word wrap default per file type (prose on, code off) with a per-panel, persisted override, replacing the single global preference.

**Architecture:** A pure helper decides the default from the path. Each editor panel carries an optional `wordWrapOverride`; the effective wrap is `override ?? default`. The floating toggle mutates that panel's override (persisted with the workspace) instead of a global setting. The git diff pane reads the default from the diffed file's path.

**Tech Stack:** React 19 + TypeScript, CodeMirror 6 (Compartment-based reconfiguration), Zustand-style workspace state persisted via tauri-plugin-store, vitest.

## Global Constraints

- Frontend checks must pass: `pnpm check-types`, `pnpm test`, `pnpm exec biome lint ./src` (RTK proxies `pnpm lint` to ESLint, which this repo does not use; call biome directly).

- Commit messages in English; never add Co-authored-by or Generated by.
- Prose types that default to wrap ON: `.md`, `.markdown`, `.mdx`, `.txt`, `.text`. Everything else OFF.
