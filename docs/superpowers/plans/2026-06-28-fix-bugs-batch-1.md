## Task 2: BUG-06 -- Fix rename staged pathspec in fallback_patch

**Files:**
- Modify: `src-tauri/src/modules/git/operations.rs` (functions `diff_inner` and `diff_content`)

**The problem:** `diff_content` calls `diff_inner` with only `rel_path` (the new name). For a staged rename, the `git diff --cached` sees the old path on one side and misses it. `commit_file_diff` already adds both paths; we mirror that here.

- [ ] **Step 1: Extend `diff_inner` to accept an optional second path**

Locate `fn diff_inner` (around line 189). Change its signature and body:

```rust
fn diff_inner(
    repo_root: &ResolvedGitDirectory,
    path: Option<&str>,
    original_path: Option<&str>,
    staged: bool,
) -> Result<GitDiffResult> {
    let mut args: Vec<OsString> = vec!["diff".into(), "--no-ext-diff".into()];
    if staged {
        args.push("--cached".into());
    }
    let pathspec = match path.filter(|p| !p.is_empty()) {
        Some(p) => Some(pathspec_from_input(&repo_root.local_path, p)?),
        None => None,
    };
    if let Some(spec) = pathspec.as_ref() {
        args.push("--".into());
        args.push(spec.clone().into());
        if let Some(orig) = original_path.filter(|o| !o.is_empty() && *o != spec.as_str()) {
            let orig_spec = pathspec_from_input(&repo_root.local_path, orig)?;
            args.push(orig_spec.into());
        }
    }
    let output = run_git(
        &repo_root.workspace,
        Some(&repo_root.git_path),
        args,
        DEFAULT_TIMEOUT_SECS,
    )?;
    ensure_success(&output, "git diff failed")?;

    let diff_text = match String::from_utf8(output.stdout) {
        Ok(text) => text,
        Err(e) => String::from_utf8_lossy(&e.into_bytes()).into_owned(),
    };
    Ok(GitDiffResult {
        diff_text,
        truncated: output.truncated,
    })
}
```

## Task 3: BUG-07 -- Fix name-status/numstat correlation using two separate invocations

**Files:**
- Modify: `src-tauri/src/modules/git/operations.rs` (function `git_commit_files` and helpers `split_name_status_numstat`, `apply_numstat`)

**The problem:** `git diff-tree --name-status --numstat -z` outputs both formats in a single stream; the heuristic that splits them by the first TAB fails for binary files (numstat shows `-\t-\t`) and some merge commits. The fix is two separate invocations correlated by path.

- [ ] **Step 1: Write a Rust test that exposes the correlation bug**

        // Binary files have "-\t-\tfilename" in numstat.
        // Verify that a text file in the same batch still gets correct counts.
        let mut files = vec![
            GitCommitFileChange {
                status: "M".to_string(),
                path: "text.rs".to_string(),
                original_path: None,
                added: 0,
                removed: 0,
            },
            GitCommitFileChange {
                status: "M".to_string(),
                path: "image.png".to_string(),
                original_path: None,
                added: 0,
                removed: 0,
            },
        ];
        // Simulate numstat: "5\t3\ttext.rs\0-\t-\timage.png\0"

        assert_eq!(files[0].removed, 3, "text file removed lines");

```rust
// Remove or keep split_name_status_numstat -- it is no longer called.
// Add a new helper:
fn apply_numstat_by_path(files: &mut [GitCommitFileChange], numstat_bytes: &[u8]) {
    // numstat -z format: "<added>\t<removed>\t<path>\0"
    // For binary: "-\t-\t<path>\0"
    let s = std::str::from_utf8(numstat_bytes).unwrap_or("");
    for token in s.split('\0').filter(|t| !t.is_empty()) {
        let parts: Vec<&str> = token.splitn(3, '\t').collect();
        if parts.len() < 3 {
            continue;
        }
        let path = parts[2];
        let added: u32 = parts[0].parse().unwrap_or(0);
        let removed: u32 = parts[1].parse().unwrap_or(0);
        if let Some(f) = files.iter_mut().find(|f| f.path == path || f.original_path.as_deref() == Some(path)) {
            f.added = added;
            f.removed = removed;
        }
    }
}
```
