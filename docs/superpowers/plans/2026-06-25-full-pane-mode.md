# Full Pane Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the existing minimal Zen Mode (header-only hide) with Full Pane Mode: F5 hides the header AND expands the active pane to fill the workspace, while all other panes remain mounted.

**Architecture:** Per-window state (`zenPaneId: string | null`) lives in `App.tsx` like the old `zenMode` boolean. It flows down as `expandedPaneId` through `WorkspaceView` → `SplitNodeView`, where leaf pane nodes get a CSS wrapper: the expanded pane uses `absolute inset-0 z-10` to break out of the flex layout and fill the workspace; all other panes get `invisible pointer-events-none` so PTYs and editors stay mounted.

**Tech Stack:** React 19, TypeScript, Tailwind v4, Vitest, `pnpm`
